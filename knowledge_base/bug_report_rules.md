# Bug Report Rules

- Заголовок должен содержать место проблемы и суть дефекта.
- Шаги должны позволять воспроизвести дефект.
- Ожидаемый результат должен быть конкретным.
- Фактический результат должен описывать реальное поведение.
- Приоритет определять по влиянию на пользователя:
  - High — блокирует ключевой сценарий.
  - Medium — мешает, но есть обходной путь.
  - Low — косметическая или незначительная проблема.
- Не смешивать несколько разных багов в одном баг-репорте.
- Не использовать слово "Sprint" в баг-репортах.
