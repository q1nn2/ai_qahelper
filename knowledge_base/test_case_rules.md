# Test Case Rules

- Один тест-кейс = одна основная проверка.
- Название должно быть конкретным и понятным.
- Шаги должны быть воспроизводимыми.
- Если есть ввод данных, указывать конкретные значения.
- Ожидаемый результат должен описывать наблюдаемое поведение.
- Не использовать общие фразы вроде "работает корректно".
- Для граничных значений указывать конкретные значения.
- Если тест зависит от другого бага или блокера, это нужно явно указать в примечании.
- Не переносить учебные названия проектов в новые документы.
- Не использовать слово "Sprint".
