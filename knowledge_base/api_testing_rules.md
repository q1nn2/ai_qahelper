# API Testing Rules

- Проверять status code.
- Проверять response body.
- Проверять обязательные поля.
- Проверять типы данных.
- Проверять schema, если она есть.
- Проверять авторизацию.
- Проверять запрос без токена.
- Проверять запрос с невалидным токеном.
- Проверять невалидные данные.
- Проверять пустые значения.
- Проверять граничные значения.
- Проверять сообщения ошибок.
- Не ограничиваться только проверкой status code.
