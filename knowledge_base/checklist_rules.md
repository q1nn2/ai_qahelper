# Checklist Rules

- Чек-лист должен покрывать основные пользовательские сценарии.
- Не дублировать одинаковые проверки.
- Группировать проверки по функциональным областям.
- Покрывать позитивные, негативные и граничные сценарии.
- Для UI проверять отображение, доступность элементов, валидацию и состояние кнопок.
- Для форм проверять обязательные поля, пустые значения, невалидные данные и ограничения длины.
- Для сложных flow проверять переходы между шагами, возврат назад, отмену действия и сохранение состояния.
