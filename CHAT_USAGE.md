# AI QAHelper Chat

Локальный чат-интерфейс для запуска `ai_qahelper` без ручного ввода CLI-команд.

## Установка

```bash
pip install -e .
```

Для работы генерации через LLM нужен настроенный `.env` или переменная окружения с API-ключом, как описано в `README.md`.

## Запуск

```bash
ai-qahelper-chat
```

Альтернативный запуск:

```bash
python -m ai_qahelper.chat_launcher
```

После запуска откроется Streamlit-интерфейс в браузере.

## Как пользоваться

В левой панели можно загрузить файлы требований `.md`, `.txt`, `.pdf` и указать `Target URL`.

В чат можно писать обычным языком:

```text
Сделай 10 тест-кейсов по загруженным требованиям
```

```text
Сделай чек-лист по требованиям
```

```text
Сгенерируй Playwright автотесты для последней сессии
```

```text
Запусти автотесты
```

```text
Создай баг-репорты по падениям pytest
```

```text
Выгрузи отчеты в Google Sheets: <ссылка на таблицу тест-кейсов> <ссылка на таблицу баг-репортов>
```

## Что умеет чат

- Определяет намерение пользователя по тексту.
- Использует существующий pipeline проекта: `agent_run`, `generate_docs`, `generate_autotests`, `run_autotests`, `draft_bugs`, `sync_reports`.
- Запоминает последнюю активную `session_id`.
- Показывает пути к созданным артефактам в `runs/<session_id>/`.
- Просит подтверждение перед действиями с внешними эффектами: запуск автотестов и выгрузка в Google Sheets.

## Ограничение

Сейчас intent-router простой и работает по ключевым словам. Если чат не понял задачу, напишите проще: `сделай тест-кейсы`, `создай автотесты`, `запусти автотесты`, `создай баги`.
